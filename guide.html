MSA Billing — Setup Guide

Is app ko apne Firebase project se jaibanane (connect) aur GitHub par host karne ki mukammal guide. Ek dafa ka setup — phir app khud chalti rahegi.

Shuru karne se pehle

Aapke paas kya hona chahiye

  • Ek Google account jisse Firebase project bana hua hai (mirza-bills — pehle se mojood).
  • Computer par Node.js 20+ (nodejs.org).
  • Ye project ki files (ye folder) computer par.
  • ~20 minute. Mostly copy-paste.

1

Cloud Firestore enable karein

Saara data (bills, products, users, settings, reports) Firestore me save hota hai.

  1. Firebase Console → Firestore Database kholein.
  2. Create database dabayein.
  3. Location: asia-southeast1 (ya jo aapke qareeb ho) — ek dafa set hone ke baad change nahi hoti.
  4. Mode: Production mode chunein (rules hum khud deploy karenge — step 6).
✓
Empty database theek hai — aap apna asli data app ke andar se daalenge.
2

Authentication enable karein

Login email + password se hota hai.

  1. Authentication → Sign-in method kholein.
  2. Email/Password ko enable karein (pehla toggle on). “Email link” ki zaroorat nahi.
  3. Save dabayein.

4

Blaze (pay-as-you-go) plan

User management ke liye Cloud Functions chahiye, jiske liye Blaze plan zaroori hai.

  1. Firebase Console → Upgrade par jayein.
  2. Blaze select karein, card add karein.
₨
Aap jaisi chhoti workshop ki usage Firebase ke free tier ke andar rehti hai — mahine ka kharcha taqreeban Rs. 0. Aap chahein to budget alert bhi laga sakte hain.
5

Firebase CLI install + login

Computer par terminal kholein aur:

# Firebase CLI install (ek dafa)
npm install -g firebase-tools

# Google account se login
firebase login

Phir is project folder me jayein (jahan firebase.json hai):

cd "raasta/is/folder/ka"
firebase use mirza-bills

Functions ki dependencies install karein

cd functions
npm install
cd ..
6

Rules, indexes & functions deploy karein

Sab kuch ek command se:

firebase deploy --only firestore:rules,firestore:indexes,functions

Ye deploy karta hai:

  • firestore.rules — security rules (permission-aware, sirf logged-in users).
  • firestore.indexes.json — bills ki sorting/filtering ke liye indexes.
  • functions/index.js — secure user-management (create/update/disable/delete/reset-password) Admin SDK se.
⏳
Pehli dafa functions deploy hone me 2–3 minute lag sakte hain. Region us-central1 hai (app isi se match karta hai).
!
Agar indexes “building” dikhayein, thoda intezar karein — bills list tab tak khali lag sakti hai. Index status yahan dekhein.
7

Authorized domain add karein

Jis domain par app host hogi, usay Firebase Auth ki authorized list me daalna zaroori hai — warna login block ho jayega.

  1. Authentication → Settings → Authorized domains.
  2. Add domain → apna GitHub Pages domain daalein, e.g. your-username.github.io.
✓
localhost aur *.firebaseapp.com pehle se allowed hote hain.
8

GitHub Pages par host karein

App khud static hai — sirf functions Firebase par chalte hain. Hosting ke do aasaan tareeqe:

Option A — GitHub Pages (jaisa aap chahte hain)

  1. Is project ko ek GitHub repo me push karein.
  2. Repo → Settings → Pages → Source: main branch, / (root).
  3. Thodi der baad URL milega: https://your-username.github.io/repo-name/.
  4. App khulay: .../index.html
ℹ︎
App khud detect kar leti hai ke wo github.io par hai aur asli Firebase backend use karti hai. (Preview/localhost par ye offline “demo” data use karti hai taake aap bina internet test kar sakein.)

Option B — Firebase Hosting (ek command)

firebase deploy --only hosting

URL: https://mirza-bills.web.app/index.html

8b

Custom domain? Set the mode explicitly

Hostname-based auto-detection (step 8's "khud detect kar leti hai") only recognizes github.io, web.app and firebaseapp.com — it has no way to know your own custom domain. Agar aap app ko apni custom domain (jaise billing.yourshop.com) par host kar rahe hain, wahan yeh detection kaam nahi karega aur app khamoshi se offline "demo" mode me chal jayegi — asli data save/load nahi hoga.

Isi liye app-config.js file hai — yahan seedha, explicit control hai:

// app-config.js
export const APP_CONFIG = {
  MODE: "production", // "production" | "demo" | "auto"
};
  • "production" — hamesha asli Firebase backend use ho (real login, real data). Custom domain ke liye yehi set karein.
  • "demo" — hamesha offline demo data (koi Firebase call nahi) — dikhane/testing ke liye.
  • "auto" — sirf jaani-pehchani Firebase hosts par auto-detect; custom domain par yeh demo mode me chala jayega.
✓
Yeh file already "production" par set hai. Agar kabhi demo dikhana ho to isay "demo" kar dein, phir dobara "production" kar dein aur redeploy karein.
ℹ︎
Quick per-browser test ke liye (bina file badle): browser console me localStorage.msa_mode = 'firebase' ya 'local' likh dein — yeh hamesha app-config.js se pehle check hota hai.
9

Pehla login & phone par install

  1. App kholein, login karein: admin@msa.com / 123456.
  2. Pehle login par admin ka profile khud ban jata hai (Asif Shakoor, saari permissions).
  3. Settings me ja kar business name, currency, bill prefix waghaira set karein.
  4. Users page se apni team add karein aur permissions dein.
  5. Phone par: browser menu → Add to Home Screen — app icon ban jayega, full-screen khulega.
🔒
Zaroori: Pehle login ke baad admin ka password ek strong password se badal dein (Users → admin → Set password).
✓
Bas! Ab app live hai. Data Firestore me mehfooz hai, offline bhi chalti hai, aur khud sync hoti rehti hai.

Aam masail

  • Login “unauthorized domain”: Step 7 (authorized domains) doobara check karein.
  • “Permission denied” / data nahi aata: rules deploy hui? user logged-in hai? (Step 6)
  • User create nahi hota: functions deploy hue? Blaze on hai? (Steps 4 & 6)
  • Bills list khali / “index building”: indexes ban rahe hain, thodi der baad.
Admin-only

Emergency access switch (kill switch)

Aap kisi bhi waqt Firestore console se saari app ki access band kar sakte hain — koi bhi user login screen tak nahi pohanch payega, sirf ek full-screen message dikhega jo skip nahi ho sakta.

  1. Firestore → app collection → access document kholein.
  2. isBlocked ko 1 kar dein — turant sab users is message par lock ho jayenge.
  3. title aur message fields edit karein (message Markdown support karta hai: # heading, **bold**, *italic*, lists, links).
  4. Wapas access dene ke liye isBlocked ko 0 kar dein.
!
Yeh sirf Firestore console (ya Admin SDK) se badalta hai — app ke andar se koi bhi is value ko set nahi kar sakta, is liye galti se koi user khud ko lock/unlock nahi kar sakta.
